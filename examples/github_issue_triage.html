AI-Crew Use Case: GitHub Issue Triage and Initial Response

This use case features an AI-Crew avatar monitoring a GitHub repository for new issues or mentions. The avatar provides initial acknowledgments, asks for necessary clarifying information (e.g., reproduction steps, logs), or points to existing documentation and guidelines using its RAG knowledge base.

Avatar Configuration Example (`profiles/issue_triage_bot.yaml`)

name: "IssueBot"
personality: "A diligent and organized GitHub bot. Focuses on gathering information and providing quick, helpful initial responses to issues. Always polite and professional."
llm_model: "llama3:8b"
llm_options:
  temperature: 0.5
  top_p: 0.8
# ... other avatar details (e.g., GitHub credentials in .env)

Prompting Strategy

The avatar's comments are generated using the github_comment_template.txt, augmented with RAG context. The prompt guides the LLM to:

  • Acknowledge the newly opened issue or mention.
  • If the issue describes a common problem, provide a direct solution or workaround from the RAG knowledge base.
  • If crucial information is missing (e.g., reproduction steps, environment details, error logs), politely ask clarifying questions to help diagnose the problem.
  • Suggest relevant documentation, contribution guidelines, or similar issues from the RAG knowledge base.
  • Maintain a professional and helpful tone.

RAG Knowledge Base Content

The RAG knowledge base (ingested into PostgreSQL with pg_vector) for this avatar would typically contain:

  • Project contribution guidelines (e.g., CONTRIBUTING.md).
  • Common error messages and their resolutions.
  • Links to official documentation, FAQs, or tutorials.
  • Information about project versions, supported environments, and known limitations.
  • Details about common feature requests or design decisions.

Example RAG context provided to the LLM: Relevant information from knowledge base: [CONTRIBUTING.md], [Known Bug: X in v1.2]
