AI-Crew Use Case: Automated Customer Support Assistant (Email-based)

This use case demonstrates an AI-Crew avatar acting as a first-line customer support agent. The avatar automatically answers common customer queries received via email, provides information retrieved from a knowledge base (RAG), and can be configured to escalate complex or unresolved issues to a human agent.

Avatar Configuration Example (`profiles/support_bot.yaml`)

name: "SupportBot"
personality: "A friendly, patient, and knowledgeable customer support agent. Always provides clear and concise answers. If unsure, asks for more details or suggests escalation."
llm_model: "llama3:8b" # Or a larger model for more complex reasoning
llm_options:
  temperature: 0.3 # Lower temperature for factual, consistent answers
  top_p: 0.7
email_history_limit: 5 # Keep recent conversation context
# ... other avatar details (e.g., email credentials in .env)

Prompting Strategy

The avatar's responses are generated using the email_reply_template.txt, augmented with conversation history and RAG context. The prompt guides the LLM to:

  • Identify the user's problem or question from the incoming email.
  • Search the provided RAG context for relevant solutions or information.
  • Formulate a polite, helpful, and concise answer based on the retrieved information.
  • If a clear answer is not found in the RAG knowledge base, the avatar is instructed to ask for more specific details or suggest escalating the issue to a human support representative.

RAG Knowledge Base Content

The RAG knowledge base (ingested into PostgreSQL with pg_vector) for this avatar would typically contain:

  • Frequently Asked Questions (FAQs) and their answers.
  • Product manuals and user guides.
  • Troubleshooting steps for common issues.
  • Company policies or service terms.
  • Information about product features and usage.

Example RAG context provided to the LLM: Relevant information from knowledge base: [FAQ: How to reset password], [Troubleshooting: Login issues]
